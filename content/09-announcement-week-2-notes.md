I hope everyone enjoyed going through the React Todo app tutorial. When looking up help it's important to know that there have been several different versions of React. A few years ago, React switched from class-based components to function-based components. Class-based components are still supported, but function-based components with hooks are now the standard. When looking through articles you may run into some examples using class-based components. Keep in mind that the code from these examples will be very different from the code you need to write.

React is a front-end client library. This week we will learn our first full stack framework, Next.js. Next.js builds on React and provides powerful features like server-side rendering (SSR), static site generation (SSG), and API routes, enabling developers to create performant and scalable applications. It's important to note that Next.js does not supply everything you need for the back-end such as a database. This week's learning activity will introduce you to Next.js. If you'd like to learn more, I'd recommend visiting their website directly:

- https://nextjs.org/

One of the most crucial elements to full stack development is learning where specific parts of the code belong. Generally, we put our business logic in the back-end as a way to enforce those business rules and keep our application secure. The front-end will generally hold display logic to help the user navigate and utilize the features of our application.

Often, we need our website to dynamically change based on user behavior. There are several ways to accomplish this.

- One way is to have the server generate new HTML code for the client to render. This is known as server-side rendering.
- Another way is to implement an API in the back-end that the client can call over HTTP. The API will return some data, and the client will dynamically change the page based on the data using JavaScript.

The advantage with this strategy is that it decouples the front-end from the back-end. This approach aligns with JAMstack principles, which emphasize pre-rendering, APIs, and JavaScript-driven interactivity. You can learn more about the JAM stack here:

- https://jamstack.org/
