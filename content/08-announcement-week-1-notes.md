## Welcome to Week 1

Welcome to Week 1 of our Full Stack Web Development course.

I'm so excited to teach you full stack web development. This is a subject that I practice every day in my full-time job. What I love about full stack development is the flexibility it gives the developer to change any part of the application without being dependent on others. It also helps you as the developer see the full picture of the application which makes it easier to plan out future enhancements and features.

Full stack development means working on anything from the front-end CSS to creating tables in the database or even provisioning servers and infrastructure to host the application.

## Technologies Used in This Course

### Front-End: React

We will be using React to build out the front-end. React is an extremely popular JavaScript library developed by Meta (Facebook). React is a highly sought after skill in the job market. Our first assignment will be creating a simple React Todo application.

### Back-End: Node.js

Next, we will be using Node.js for our back-end development. Node.js is a very popular runtime, mainly because it uses JavaScript as its language. This gives developers a huge advantage where they don't have to context switch between languages when working between the front-end and back-end.

There are several web servers you can use inside of Node.js. Express, NestJS, and Koa being popular options.

### Full Stack Framework: Next.js

Next, we will be using Next.js as our full stack framework. Next.js is a powerful React framework designed for building fast and scalable full-stack web applications. 

**Key features:**

- Server-side rendering (SSR)
- Static site generation (SSG)
- API routes
- Built-in support for dynamic routing
- Automatic optimizations: image optimization and code splitting
- Seamless integration with backend services

Next.js enhances performance while simplifying deployment, making it a great choice for modern web applications that require both client-side interactivity and server-side functionality.

### Deployment: Vercel

Next, we will be using a serverless technology called Vercel. Vercel Serverless Functions provide an easy way to deploy backend logic without managing infrastructure.

**How it works:**

- Built on AWS Lambda
- Automatically scales based on demand
- Supports Node.js, Go, Python, and other runtimes
- Enables dynamic API routes
- Handles requests efficiently
- Integrates seamlessly with front-end frameworks like Next.js

Developers can focus on writing code without worrying about server provisioning, scaling, or maintenance, making them ideal for modern, cloud-native applications.

### Version Control: GitHub

We will be utilizing GitHub as our version control repository. You will need to make sure to use a public repository so you can share the URL with the graders.

**Important notes:**

- If you don't already have a GitHub account, you will need to sign up for one.
- If you have not used Git before, I would recommend running through a quick tutorial on YouTube.
- Make sure all your code changes have been merged into your Main branch before you submit.
